# __Interface__

__Interface__ in C# is kind of making rules and regulations to classes.<br />
The _interface_ is inheritable just like the parent class but it works differently.<br />
We will see how different the interface is and when to use that in this blog post.

<hr />

<br />

What should we do after we declared an interface to use them appropriately?<br />
You hae to inherit that interface and implement all the methods that already declared in interface.<br />
If you do not implement all the methods, then you get an compilation error.
Interface is forcing the class to make some specific methods.

> __IMPORTANT:__<br />
> You have to use _public_ when you implement the interface methods.<br />

<hr />

### __Interface cannot make any Instances__

Obviously, the main job of class is determining what kind of data and function the instance should have.<br />
When it comes to interface, it cannot define any statuses or behaviours of instances because its main purpose is just making rules for classes.<br />
For that reason, C# initially banned the creation of instances through the interfaces.<br />

 - You can implement methods in interface.
 - If you do that, then you do not have to redeclare them in derived class.
 - In case you want to access them, you have to upcast to invoke them.
